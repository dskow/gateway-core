# Security Policy

## Supported Versions

The following table outlines which versions of core-gateway currently receive security updates. Only actively maintained branches will receive patches for vulnerabilities.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously and appreciate responsible disclosure. If you believe you’ve found a security vulnerability in core-gateway, please follow the process below.

🔐 How to Report

Please email your report to:

security@dskow.com
(or your preferred contact method—GitHub Security Advisories, private issue, etc.)

Include as much detail as possible:

• A clear description of the issue
• Steps to reproduce
• A proof-of-concept (if available)
• The potential impact
• Any suggested fixes or mitigations


⏱ Expected Response Timeline

We aim to respond according to the following schedule:

• Initial acknowledgment: within 48 hours
• Triage and severity assessment: within 5 business days
• Status updates: at least once every 7 days until resolution
